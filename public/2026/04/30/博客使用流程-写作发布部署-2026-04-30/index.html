六、上线后验证

部署完成后请检查：

  • 线上首页是否出现新文章。
  • 文章详情页链接是否正确。
  • 手机端页面是否可正常访问。

七、常见问题

1）npm run build 失败

优先检查：

  • 依赖是否安装完整（重新执行 npm install）。
  • 主题目录是否完整（当前项目配置为 fluid）。

2）部署成功但线上没更新

优先检查：

  • 是否推送到了正确仓库与分支。
  • GitHub Pages 是否已完成构建（通常有几分钟延迟）。

3）文章时间不对

优先检查：

  • Front Matter 中 date 是否按本地时区填写。
