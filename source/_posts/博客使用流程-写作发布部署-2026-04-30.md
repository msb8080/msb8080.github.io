这篇文章给博客用户一个可直接执行的流程：从写文章，到本地预览，再到上线发布。

## 一、准备环境

先确认本机已经安装：

```bash
node -v
npm -v
```

进入博客项目根目录后，安装依赖：

```bash
npm install
```

## 二、写博客

### 1）创建新文章

```bash
npx hexo new post "我的第一篇正式博客"
```

执行后会在 `source/_posts/` 下生成一个新的 Markdown 文件。

### 2）编辑文章内容

打开新文件，使用以下结构：

```markdown
---
title: 我的第一篇正式博客
date: 2026-04-30 14:00:00
tags:
  - 技术
  - 经验
categories:
  - 学习笔记
---

这里写正文。
```

建议：
- 标题明确，不要过长。
- `tags` 用于关键词检索。
- `categories` 用于栏目归档。

## 三、发布博客（生成站点）

每次发布前，先清理再生成：

```bash
npm run clean
npm run build
```

执行后会更新 `public/` 目录，里面就是最终静态页面。

## 四、本地预览（上线前检查）

```bash
npm run server
```

浏览器访问：

```text
http://localhost:4000
```

重点检查：
- 新文章是否出现在首页。
- 文章链接是否可访问。
- 图片、代码高亮、目录是否正常。

## 五、部署上线

确认内容无误后执行：

```bash
npm run deploy
```

该命令会把 `public/` 推送到配置的 GitHub Pages 仓库分支。

## 六、上线后验证

部署完成后请检查：
- 线上首页是否出现新文章。
- 文章详情页链接是否正确。
- 手机端页面是否可正常访问。

## 七、常见问题

### 1）`npm run build` 失败
优先检查：
- 依赖是否安装完整（重新执行 `npm install`）。
- 主题目录是否完整（当前项目配置为 `fluid`）。

### 2）部署成功但线上没更新
优先检查：
- 是否推送到了正确仓库与分支。
- GitHub Pages 是否已完成构建（通常有几分钟延迟）。

### 3）文章时间不对
优先检查：
- Front Matter 中 `date` 是否按本地时区填写。

## 八、推荐日常流程（最简版）

```bash
# 1. 写作
npx hexo new post "文章标题"

# 2. 本地检查
npm run clean
npm run server

# 3. 发布上线
npm run build
npm run deploy
```

按这个流程执行，可以稳定完成“写作 -> 预览 -> 上线”的完整闭环。
